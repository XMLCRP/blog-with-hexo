1. 数据库表结构设计

2. 创建数据表

3）连接数据库

  • 连接参数：
    ◦ 本地连接使用localhost
    ◦ 默认端口3306
    ◦ 用户名root
    ◦ 密码123（示例）
  • 验证方式：通过测试连接确认配置正确
  • 版本显示：连接成功后会显示MySQL版本（如8.0.18）

4）创建数据表两种方式

  • SQL语句方式：
    ◦ 在查询编辑器编写CREATE TABLE语句
    ◦ 支持完整的SQL语法
  • 可视化方式：
    ◦ 右键菜单创建数据库/表
    ◦ 图形化设置字段类型、长度等属性
    ◦ 实时生成SQL预览
  • 操作提示：创建完成后有确认提示，可选择继续创建或结束

5）导入数据库文件

  • 文件格式：.sql后缀的文本文件
  • 导入步骤：
    ◦ 右键数据库选择”执行SQL脚本”
    ◦ 选择脚本文件路径
    ◦ 执行后刷新对象浏览器
  • 导出操作：
    ◦ 右键数据库选择”备份/导出”
    ◦ 可选择仅结构或包含数据
    ◦ 生成标准SQL文件便于迁移
    ◦
  • 导出选项：
    ◦ 包含CREATE DATABASE语句
    ◦ 设置FOREIGN_KEY_CHECKS=0
    ◦ 批量INSERT语句优化
    ◦ 文件命名：建议包含日期/版本标识

3. 知识小结

知识点 核心内容 关键操作/注意事项
数据库表设计 设计了四个表：用户表（权限管理）、产品表（生产数据记录）、统计表（按天聚合数据）、日志表（系统日志报警） 表关系设计 需符合业务需求，统计表用于优化查询性能
MySQL客户端工具 推荐使用 SQLyog（收费但有授权），支持可视化操作与 SQL 脚本执行 安装时注意选择 64 位版本，授权信息在安装必读 TXT 中
数据库创建方式 1. 通过 SQL 语句编写（如 CREATE TABLE）；2. 通过客户端右键菜单直接创建 可视化操作 更直观，适合初学者
数据库导入/导出 导入：执行 SQL 脚本文件（.sql 后缀）；导出：备份为 SQL 文件（含结构或数据） 导出选项 需明确选择“仅结构”或“包含数据”
实战操作演示 1. 连接本地 MySQL（主机名 localhost，端口 3306）；2. 创建字段时指定类型（如 VARCHAR(100)） 测试连接 确保配置正确，字段类型需匹配实际数据
