  3. （主窗体）创建委托实现方法

4. 总结

知识点 核心内容 考试重点/易混淆点
委托技术实现窗体交互 通过委托实现跨窗体调用（FM navy → FM main），包含委托声明、对象创建、方法体实现、绑定及调用五步骤 委托绑定逻辑（openForm 方法体与委托对象关联）
窗体重复打开判断 使用 FindDocument 方法检查窗体是否已存在，避免重复实例化 docContent 对象判空逻辑与 BringToFront() 调用时机
动态窗体加载 根据按钮参数（如“集中监控”“系统设置”）实例化对应窗体（FM_monit / FM_set） 窗体类需继承 docContent 基类
委托调用流程 openForm 委托触发 → 匹配窗体名称 → 激活或新建窗体 参数传递一致性（窗体名称字符串单匹配）
界面设计核心 主窗体（docMain）与子窗体的容器关系管理 Documents 集合的历史与控件激活
